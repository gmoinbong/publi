## Проверка после деплоя:

### 1. Проверь структуру на FTP:
```
/public_html/ (или ваш FTP_PATH)
├── .htaccess          ← ДОЛЖЕН БЫТЬ!
├── index.html
├── assets/
│   ├── index-*.js
│   └── style-*.css   ← ДОЛЖЕН БЫТЬ!
└── images/
    └── *.png, *.svg
```

### 2. Проверь в браузере (DevTools → Network):

**CSS файл:**
- Статус: `200 OK`
- Content-Type: `text/css` или `text/css; charset=utf-8`
- Размер: ~24-25 KB (не пустой!)

**Если 404:**
- Проверь путь в `index.html`: должно быть `./assets/style-*.css`
- Проверь, что файл существует на FTP

**Если 200, но пустой или неправильный MIME:**
- Проверь `.htaccess` на сервере
- Проверь, что Apache поддерживает `mod_mime` и `mod_headers`

### 3. Проверь содержимое CSS:

Открой `assets/style-*.css` в браузере. Должно начинаться с:
```css
/*! tailwindcss v4.1.17 | MIT License | https://tailwindcss.com */
```

И содержать классы типа:
- `.flex`, `.grid`, `.text-center`
- `.bg-white`, `.text-black`
- И т.д.

### 4. Если стили все еще не работают:

**Вариант A: Проблема с кэшем**
```bash
# Hard refresh в браузере
Ctrl+Shift+R (Windows)
Cmd+Shift+R (Mac)

# Или открой в режиме инкогнито
```

**Вариант B: Проблема с путями**
- Убедись, что `base: './'` в `vite.config.ts`
- Проверь, что в `index.html` пути относительные

**Вариант C: Проблема с хостингом**
- Некоторые хостинги не поддерживают `.htaccess`
- Попробуй другой хостинг или свяжись с поддержкой

**Вариант D: Проблема с Tailwind v4**
- Если хостинг не поддерживает новые фичи CSS
- Можно откатиться на Tailwind v3 (но это не рекомендуется)

## Локальная проверка:

```bash
cd frontend
npm run build
npm run preview
```

Открой http://localhost:4173 - если там стили работают, проблема точно на хостинге.

## Быстрая диагностика:

1. Открой сайт → F12 → Network
2. Найди `style-*.css`
3. Кликни на него → вкладка "Headers"
4. Проверь:
   - `Content-Type: text/css`
   - `Status Code: 200`
5. Вкладка "Response" → должен быть CSS код

Если все это ОК, но стили не применяются - проблема в браузере/кэше.
